Get Started Documentation

File-based Routing

Your file structure IS your CLI structure. Drop .zig files in a directory and zcli generates the routing.

Type-safe Parsing

Arguments and options validated at compile time. Your Args and Options structs define what's valid.

Zero Runtime Cost

All command discovery and code generation happens at build time. No reflection, no dynamic dispatch.

Plugin System

Extensible architecture with lifecycle hooks, global options, and build-time configuration.

Quick Install

curl -fsSL https://raw.githubusercontent.com/ryanhair/zcli/main/install.sh | sh

Then create your first CLI:

zcli init myapp
cd myapp
zig build
./zig-out/bin/myapp --help

Why zcli?

zcli uses convention over configuration. Your file structure directly maps to your CLI commands:

src/commands/
├── hello.zig        # → myapp hello <name>
├── users/
│   ├── list.zig     # → myapp users list
│   └── create.zig   # → myapp users create <email>
└── config/
    └── set.zig      # → myapp config set <key> <value>

No routing code. No boilerplate. Just drop .zig files and build.
